## Design Context

### Users
- Primary users are visual thinkers who want to remember life and work moments through images instead of long text notes.
- Typical contexts include quick capture on desktop/mobile, collecting screenshots or photos, and later recalling details with minimal friction.
- The core job-to-be-done is: capture visual moments fast, then retrieve them by intent or memory cues.

### Brand Personality
- Personality keywords: practical, vivid, trustworthy.
- Emotional goals: confidence during capture, clarity during search, and relief when recalling.
- Product voice should feel direct and useful, not decorative-first.

### Aesthetic Direction
- Direction: editorial-functional with warm highlights.
- Keep orange as the strongest accent and use soft, tinted neutrals for structure.
- Prefer clear hierarchy, asymmetric composition, and lightweight atmospheric backgrounds.
- Avoid generic SaaS clichés: sterile monochrome cards, weak hierarchy, or template-like icon blocks.

### Design Principles
1. Speed-first interactions: every screen should prioritize quick capture and predictable follow-up actions.
2. Visual memory over text complexity: imagery and context cues should be easier to scan than prose.
3. Distinctive but restrained identity: one strong accent and clear typography, without over-decoration.
4. Readability before ornament: concise copy, strong contrast, and intentional spacing rhythm.
5. Mobile reliability: preserve hierarchy and tap comfort on small screens without feature loss.

### Implementation Guardrails
- Keep visual changes composable: prefer shared component classes and small overrides instead of page-level one-offs.
- For floating surfaces (notifications, user menu, action menus), use one elevation primitive (`elevated-popover`).
- For async actions, show local state transitions near the trigger; avoid success toast for mere request acceptance.
- Never break method-based links (`data-method`) with custom navigation interception.

### Output Discipline
- If introducing new UI patterns, update `docs/guides/coding/uiux.md` in the same change.
- Keep this file high-level and intent-focused; do not duplicate low-level class rules here.
