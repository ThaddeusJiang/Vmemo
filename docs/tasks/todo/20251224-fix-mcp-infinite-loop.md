# 20251224 修复 MCP 死循环问题

## 任务目标

修复 chatbot 调用 MCP 时服务器陷入死循环的问题。每次调用 MCP 都会不断重复查询数据库，导致服务器资源浪费。

## 问题分析

### 错误现象

从日志来看：

1. GET /mcp 请求
2. 查询 `api_tokens` 表（验证 token）
3. 查询 `ash_users` 表（加载用户信息）
4. 返回 `Chunked 200 in 4ms`
5. **不断重复上述过程**

### 可能的原因

1. **MCP 客户端不断轮询**：客户端可能错误地不断发送请求
2. **流式响应未正确关闭**：MCP 协议可能使用 Server-Sent Events (SSE)，响应流没有正确关闭
3. **认证逻辑问题**：每次请求都重新验证 token，但这不是问题本身
4. **路由配置问题**：`forward "/"` 可能导致路由匹配问题

## 计划阶段

### 调查步骤

1. **检查 MCP 路由配置**：查看 `AshAi.Mcp.Router` 的使用方式
2. **检查认证逻辑**：确认 `McpAuth` 是否正确处理请求
3. **查看日志模式**：分析日志中的请求模式，确定是客户端问题还是服务端问题
4. **检查流式响应**：如果 MCP 使用流式响应，检查是否正确关闭连接

### 解决方案

根据调查结果，可能的修复方案：

1. 修复路由配置
2. 修复认证逻辑
3. 修复流式响应处理
4. 添加请求限流或去重

## 执行记录

### 阶段一：问题分析

- **时间**：20251224
- **操作**：分析日志和代码结构
- **结果**：
  - 发现每次 GET /mcp 请求都会触发认证逻辑（`McpAuth`）
  - `handle_get` 函数发送 SSE 事件后调用 `halt()`，但连接可能没有正确关闭
  - 客户端可能不断重试 GET 请求，导致死循环

### 阶段二：问题定位

- **时间**：20251224
- **操作**：检查 `AshAi.Mcp.Router` 和 `AshAi.Mcp.Server` 的实现
- **结果**：
  - `handle_get` 函数用于 SSE endpoint 发现，发送 endpoint 事件后调用 `halt()`
  - GET 请求仅用于获取 endpoint URL，不需要认证
  - POST 请求才需要认证（用于 JSON-RPC 消息）

### 阶段三：修复问题（第一版）

- **时间**：20251224
- **操作**：修改 `McpAuth`，跳过 GET 请求的认证
- **结果**：
  - GET 请求不再触发认证逻辑，避免数据库查询死循环
  - POST 请求仍然会进行认证（如果需要）
  - **但问题依然存在**：客户端仍然不断发送 GET 请求

### 阶段四：问题分析（第二版）

- **时间**：20251224
- **操作**：分析客户端不断重试的原因
- **结果**：
  - 数据库查询已消失，说明第一版修复有效
  - 但 GET 请求仍在不断重复
  - 可能是客户端（chatbot）配置问题，或 SSE 连接未正确关闭

### 阶段五：添加请求去重机制

- **时间**：20251224
- **操作**：在 `McpAuth` 中添加请求去重机制，防止客户端在短时间内重复发送 GET 请求
- **结果**：
  - 使用 ETS 表跟踪最近的 GET 请求
  - 如果同一 IP + 路径在 5 秒内重复请求，返回 429 Too Many Requests
  - 可以有效防止客户端快速重试导致的死循环

## 测试记录

- [待测试] 需要验证修复后的效果
- 预期行为：
  - 第一次 GET 请求正常处理
  - 5 秒内的重复请求返回 429 错误
  - 日志中会显示 "MCP GET rapid retry detected" 警告

## 总结

- **问题原因**：

  1. GET 请求用于 SSE endpoint 发现，不需要认证，但 `McpAuth` 对所有请求都进行认证，导致客户端不断重试时触发数据库查询死循环
  2. 客户端（chatbot）可能配置有问题，或 SSE 连接未正确关闭，导致不断重试 GET 请求

- **解决方案**：

  1. **第一版**：在 `McpAuth` 中跳过 GET 请求的认证，只对 POST 请求进行认证（解决了数据库查询死循环）
  2. **第二版**：添加请求去重机制，使用 ETS 表跟踪最近的 GET 请求，防止客户端在短时间内重复请求（解决了请求死循环）

- **修复文件**：`lib/vmemo_web/mcp_auth.ex`
