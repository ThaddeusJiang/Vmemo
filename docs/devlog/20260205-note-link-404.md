## 背景
- 页面：`/photos/:id` 中点击 note 链接跳转到 `/notes/:id`，返回 404。
- 现状：Note 页面数据来源于 Typesense，若同步作业未完成或索引缺失，会直接渲染 not_found。

## 诊断
- `VmemoWeb.NoteIdLive` 使用 `TsNote.get/2`（Typesense）读取 note 与 photos。
- 按项目规范，数据库应为实时数据源，Typesense 通过 Oban 异步同步；当前实现违反规范，索引缺失即导致 404。
- `NoteUpdateForm` 也仅操作 Typesense，未更新数据库，进一步加剧不一致。

## 方案
- 读路径：改为使用 Ash 资源 `Vmemo.Photos.Note` 从数据库读取并 `load: [:photos]`，actor 为当前用户，缺失时回退为空状态。
- 写/删路径：在组件中改为使用 Ash `:update` 和 `:destroy` 动作，保留表单错误并不丢失输入。
- 传递 actor：将 `current_ash_user` 传入组件，以便 Ash 授权。

## 变更
- `lib/vmemo_web/live/note_id_live.ex`
  - 切换到 `Ash.get(Note, id, load: [:photos], actor: current_ash_user)`。
  - 将 `current_ash_user` 传递给 `NoteUpdateForm`。
- `lib/vmemo_web/live/components/note_update_form.ex`
  - 改为 Ash update/destroy，并在失败时保持表单值与错误。
  - 更新成功后重新加载 photos，保持页面数据一致。
