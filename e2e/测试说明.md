# Vmemo E2E 自动化测试文档

## 概述

本项目使用 Playwright 实现了 Vmemo 应用的端到端自动化测试，包含登录功能和图片上传功能的完整测试用例，并支持视频录制功能。

## 功能特性

- ✅ 用户登录功能测试
- ✅ 图片上传功能测试（单张和多张）
- ✅ 所有测试自动录制视频
- ✅ 关键操作自动截图
- ✅ 自动生成测试报告

## 测试环境要求

- Node.js (v16 或更高版本)
- 运行中的 Vmemo 应用
- 测试用户账号：
  - 邮箱: `test@example.com`
  - 密码: `password123456`

## 安装步骤

```bash
cd e2e
npm install
npx playwright install chromium
```

## 运行测试

### 运行所有测试
```bash
npm test
```

### 以可视化模式运行（可以看到浏览器）
```bash
npm run test:headed
```

### 以交互式 UI 模式运行
```bash
npm run test:ui
```

### 以调试模式运行
```bash
npm run test:debug
```

### 运行特定测试文件
```bash
npx playwright test tests/login.spec.js      # 只运行登录测试
npx playwright test tests/upload.spec.js     # 只运行上传测试
```

## 测试用例说明

### 登录功能测试 (login.spec.js)

1. **成功登录测试**
   - 使用有效凭据登录
   - 验证登录后跳转到首页
   - 自动截图保存

2. **登录失败测试**
   - 使用无效凭据登录
   - 验证错误提示显示
   - 自动截图保存

3. **导航到注册页面**
   - 点击"注册"链接
   - 验证跳转到注册页面

4. **导航到忘记密码页面**
   - 点击"忘记密码"链接
   - 验证跳转到密码重置页面

### 图片上传功能测试 (upload.spec.js)

1. **单张图片上传测试**
   - 登录后导航到上传页面
   - 选择单张图片上传
   - 添加备注信息
   - 验证上传成功并跳转

2. **多张图片上传测试**
   - 同时选择多张图片上传
   - 添加备注信息
   - 勾选"Is whole"选项
   - 验证批量上传成功

3. **取消上传测试**
   - 选择图片后取消上传
   - 验证图片预览被移除

4. **表单元素验证测试**
   - 验证上传表单各元素正确显示
   - 验证拖拽区域、备注框、上传按钮等

## 视频录制

所有测试运行都会自动录制视频。测试完成后，可以在以下位置找到：

- **视频文件**: `test-results/` 目录（每个测试一个视频）
- **截图文件**: `test-results/` 目录
- **HTML 报告**: 运行 `npx playwright show-report` 查看详细结果

### 视频文件位置示例
```
test-results/
├── login-User-Login-should-successfully-login-chromium/
│   └── video.webm
├── upload-Photo-Upload-should-upload-single-image-chromium/
│   └── video.webm
└── ...
```

## 配置说明

### Playwright 配置 (playwright.config.js)

主要配置项：
- **视频录制**: `video: 'on'` - 为所有测试录制视频
- **截图**: `screenshot: 'on'` - 在关键操作时截图
- **基础 URL**: `http://localhost:4000` (可通过 `BASE_URL` 环境变量配置)
- **Web 服务器**: 自动启动 Phoenix 服务器（如果未运行）

### 修改视频录制设置

编辑 `playwright.config.js`：

```javascript
// 只在失败时录制
video: 'retain-on-failure'

// 关闭视频录制
video: 'off'

// 始终录制（默认）
video: 'on'
```

## 常见问题

### 测试失败：找不到元素
- 确保应用运行在 `http://localhost:4000`
- 检查测试用户是否存在于数据库中
- 验证选择器是否匹配应用的 HTML 结构

### 视频文件未生成
- 检查 `test-results/` 目录
- 确保 `playwright.config.js` 中设置了 `video: 'on'`
- 即使测试通过，视频也会被保存

### 应用无法启动
- 手动启动应用：`cd .. && docker compose up -d && mix phx.server`
- 或在 `playwright.config.js` 中禁用 webServer 配置

## CI/CD 集成

在 GitHub Actions 中运行测试：

```yaml
- name: 安装依赖
  run: cd e2e && npm install

- name: 安装 Playwright 浏览器
  run: cd e2e && npx playwright install --with-deps chromium

- name: 运行测试
  run: cd e2e && npm test

- name: 上传测试结果
  if: always()
  uses: actions/upload-artifact@v3
  with:
    name: playwright-results
    path: e2e/test-results/
```

## 扩展测试

### 添加新的测试用例

1. 在 `tests/` 目录创建新的 `.spec.js` 文件
2. 使用 Playwright 的 API 编写测试
3. 运行测试验证

示例：
```javascript
const { test, expect } = require('@playwright/test');

test.describe('新功能测试', () => {
  test('测试用例描述', async ({ page }) => {
    await page.goto('/your-page');
    // 添加测试步骤
    await page.screenshot({ path: 'test-results/your-screenshot.png' });
  });
});
```
